Export as Web Maps

Overview

When you export a map using the 'Export as Web Maps' option, a html page will be built which has an expandable menu down the left-hand side of the screen listing all the views and nodes that you chose to export. For each map view, an image map is produced, (which is displayed in the main part of the screen).

As maps are jpegs of the Compendium maps exported they appear exactly as they do in Compendium. You can roll over nodes to view a detail preview, click on nodes to open their detail windows in full, roll over Tag indicators to view Tags, roll over Transclusion indicators to see a clickable list of a node's parent views.

This export option now also has export to full depth capabilities, as well as the new options to export with references and to a zip file.

Export Options

When you select to export to Web Maps you will see the following dialog:


Figure 1. The Web Maps Export dialog

Here you specify which views and nodes you wish to export using the following options:


Option

Purpose

Current View Only

If this option is selected, the view to export is considered to be the current view only.

Selected Views

If this option is selected, the views which will be exported will be any views that you have selected on the current map.

Other Views

If this option is selected, the 'Choose Views' button will become available. If you press this button a dialog will open similar to the one in figure 2 below. This dialog will list all the views in your current database so you can select the ones you wish to export.

Nodes in view only

If this option is selected, each view will have all its nodes exported.

One level down

If this option is selected, each view will have its nodes exported and if any of them are map or list nodes, it will export their nodes as well.

Full depth

If this option is selected, each view will have its nodes exported and if any of them are map or list nodes, it will export their nodes as well, and if any of their nodes are maps or lists, is will export their nodes as well etc.... (depending on you view relationships, this options could potentially export many or all of your views, so use it with caution!).

HTML title for the base web page

Any text typed in this field will be used as the HTML page title for the main web page (as shown in the Browser title bar).
Exclude detail popups which only contain the node label & anchor id? If you don't want detail popups to be created if there are no node details apart from the node label and anchor id, then select this.
Exclude detail popups altogether? If you don't want detail popups to be created at all.
Open references and Images in a new window? This allows you to decide if you want references and images than you click on to view opened in a new Window or in the same window.
Add titles to exported views If you select this then a map will have the map title (the node label for the map) added to the top of each map page.
List Views Alphabetically on Menu? If you select this option the View listed on the Left side menu pane will be listed alphabetically, instead of logically.
Include reference files? If you select this option any locally referenced files will be included in the export in a directory called 'references'.
Export to Zip Archive? If you select this option, all the files and folder generated will be placed inside a single zip file
Open Export after completion? If you select this, the HTML page will be opened in your default browser after the export has completed.

Figure 2. The views selection dialog.


Exporting

Once you have set any export options you wish to, you press the 'Export' button (see figure 1 and 2). This will open a file browser window, similar to the one in figure 3, below.


Figure 3. The Export File Browser dialog.


You then need to specify the file you wish to export to, either a html file for the top file, or a zip file if exporting to zip, and then press 'Save'. The export process will then begin.
