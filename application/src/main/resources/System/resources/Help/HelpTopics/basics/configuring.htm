Configuring Compendium

There are several ways that you can configure Compendium for your own or your Company's needs. These options are for the more advanced users only and are quite technical.

  • The System.ini file
  • Reference Node Configuration
  • Adding Icons Sets (Skins)

Modifying the System.ini file and related data

Compendium 2.0 introduced a new system.ini file system to configure certain aspects of Compendium externally. The system.ini file can be found at <compendium dir>/System/ini/

There are three things in this folder by default:
  1. system.ini file
  2. images folder: holding all the images referenced by the system.ini file
  3. DefaultData.xml file: holding the default data loaded into a new Project in Compendium. This file is a Compendium XML Export file.

The system.ini file contents

This file is a property file consisting of a long list of option=value pairs. There is specific documentation on each option=pair inside the file.


General Options

applicationNameAddition=
Text to place after the application name to qualify a version or flavour of the application. A single space will be inserted between the Application name and this addition. e.g. if this is set to 'eDance' the application name will become 'Compendium eDance'.

projectDefaultDataFile=System/ini/DefaultData.xml
Specify the XML file to use to load the default data for a project. You can choose to leave this blank if you want no default data at all. This file must be a Compendium xml export file (or a Compendium XML zip file). We recommend that you located any referenced files in your default data in the ini folder. If you choose to use a Compendium xml zip file be aware that the linked file resources will then be copied into each new Project's LinkedFiles folder rather than having one set of referenced files centrally for all projects. The choice is yours, but we would recommend you use an xml file with centrally referenced external resources, if required.

createDefaultProject=false
When Compendium first runs, should it create a first default project automatically? The default value if 'false'. Change this to 'true', if you want Compendium To create a default project after the user has installed it.
Note: see also 'defaultProjectName' below.

defaultProjectName=
If you are letting Compendium auto-create the first default project, this is the name that project will be given.
see also 'createDefaultProject' above.

defaultHomeViewBackgroundImage=
Specify a default background image to be used on all user home views

defaultMapBackgroundImage=
Specify a default background image to be used on all maps. Useful when company water marks etc are required.

defaultStencilSetName=
If this is set it will open this stencil set by default.
Note: the name below must match a stencil set name exactly for it to open it.

bannerImage=
>The banner image file for image that sits under the menu bar when the Compendium application is running.
The image file should be placed in System/ini/images folder.
The suggested image size is 4500w x 18h. It will be loaded 0x,0y under the Menubar.

quickStartMovie=System/resources/Help/Movies/welcome.html
This is the link on the Help menu for the Quick start Movie.
It can be a local file path or a URL

releaseNotesURL=http://compendium.open.ac.uk/institute/download/release-notes-2.0-beta1.htm
This is the link on the Help menu for the release notes for the current version of Compendium.
It can be a local file path or a URL

defaultPowerExportPath=Exports
The path that a Power Export should go into by default.
This was made editable as the Open Learn version had a subfolder in Exports for these specific exports.
The default is just the main 'Exports' folder. If left blank, that is where the Power Export will go to.

simpleInterface=true
The interface mode to start Compendium with when first installed.
'true' = simplified interface; 'false' = full interface.
The user can override this and their choice is stored in Format.properties


Startup Splash / About Window Options

startUpTitle=Starting <appname>
Splash dialog title when Compendium starts. Can tailor this to specify a version
like 'Starting Compendium - OpenLearn' or 'Starting CompendiumLD'
If you want the Compendium application name included in the message
add <appname> where you want it to appear.

Text which appears under the title on the start up screen.
Should be a short sentence.
You can make Compendium display two lines of text at this point by entering it
here as HTML e.g. <html>A Tool for the Compendium<br>Methodology</html>

Should the admin database name be displayed in the startup screen?
'true' = yes; 'false' = no.

The startup/about dialog splash background image file.

Reference Node Configuration

You can modify or add to the icons that are drawn for different file types or urls on a reference node.
Go to the <compendium dir>/System/resources/ReferenceNodeIcons folder.

Inside this folder you will see all the icons for the file types that Compendium recognizes by default. There will also be an xml file called: referencenodetypes.xml. This file defines the rules for Compendium to associated references with icons. Not only can you use this file to add your own types, you can rearrange the process order by moving the order of the sections inside the file.
For a general basic understanding of the file there is a DTD at the top of it, but basically the rules are as follows:

reference_node_type

An outer 'reference_node_type' block with required attributes of:
  • name= a unique name for this reference type
  • icon= the large icon used (usually 32x32, but can be any size)
  • icon_small= the small icon used (usually 16x16, but can be any size)

match_group

Inside the 'reference_node_type' block you can have ONE or MORE 'match_group' blocks. This has only one required attribute of 'appended_operator'. The values for this can be either 'none', 'and' or 'or'. The way this operator acts is to join together either internal match_term items, or if there is only one internal item, this match group to the next match group.
For example:
	<match_group appended_operator="or">
            <match term="www" type="starts_with" />
            <match term="http:" type="starts_with" />
            <match term="https:" type="starts_with" />
   	</match_group>

Here the Reference type is matched if either the reference string starts with www or http or https.
And for the full block of this example: 
      <match_group appended_operator="and">
            <match term="labspace.open.ac.uk" type="contains" />
      </match_group>
      <match_group appended_operator="or">
            <match term="www" type="starts_with" />
            <match term="http:" type="starts_with" />
            <match term="https:" type="starts_with" />
      </match_group>
Here the Reference type is matched if the reference string contains "labspace.open.ac.uk" and either starts with www or http or https.

match_term

Inside each match_group is one or more match_term items. Each match_term item has two required attributes:
  1. term=the string to match or the string length number
  2. type=the type of match to perform. Here there are the following options:
    • starts_with = test is true if the reference string starts with the current term
    • ends_with   = test is true if the reference string ends with the current term
    • contains    = test is true if the reference string contains the current term
    • length_equals = test is if the length of the reference string equals the term
    • length_less = test is if the length of the reference string is less that the term
    • length_more = test is if the length of the reference string is greater than the term
    • isDirectory = test is true if the reference string is the path to a directory (the term in this case is an empty string)

Before you edit the referencenodetypes.xml file, it is best to make a copy of the original file first.



Adding Icon Sets (Skins)

Icon skins are basically the icon images used for the Compendium main node set. These are what appear on the node creation toolbar and what are used when you create nodes with the shortcut keys and right-click menu.

Compendium comes with several sets of skins for the user to choose from. Many are there for historical reasons. The icon set that will be chosen by default, is called 'Default'. You can change which icon set Compendium uses from Tools Menu / User Options / Misc / Icon Sets. As well as choose from a set of skins that comes with Compendium, you could replace the icons used with your own set. Note: this is different from creating your own Stencil Sets.

In the main Compendium folder is a sub-folder called 'Skins'. Inside this is a list of folders, one for each icon set that comes with Compendium. The main icon set used in Compendium is called 'Default'. The folder names are used as the option names in the User Options window.

If you want to create a new skin, add a folder inside the Skins folder and give it a name (remember the folder name will be used in the icon set menu, so label carefully!). Inside this folder there needs to be a set of images that exactly match the file names of the image sets in all the other Skins folders. They can either be gif files or png files. We would recommend png files.
